## 클린 코드가 강조하는 것

- 읽기 좋은 코드
- 이해하기 좋은 코드
- 확장하기 좋은 코드
- 유지보수하기 좋은 코드

결국, 유지보수성!

그럼 클린 아키텍처는?

품질에 관한 요구사항(비기능적 요구사항)에서 유지보수성은 특별하다
유지보수하기 좋은 코드 : 확장하기 좋고, 안전하고, 신뢰할 수 있고, 좋은 성능으로 발전시킬 수 있고, 상호 호환성이 뛰어나서 변경하기 쉽다.
유지보수성 : 코드의 변경가능성과 동의어
